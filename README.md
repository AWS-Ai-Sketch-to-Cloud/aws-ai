# 🎨 Sketch-to-Cloud

### “손그림 한 장으로 끝내는 AI 클라우드 설계 비서”

---

# 1. 프로젝트 개요 (Overview)

Sketch-to-Cloud는

클라우드 인프라 지식이 부족한 사용자도 **스케치 또는 텍스트 입력만으로 AWS 인프라를 설계하고, 비용을 예측하며, 코드까지 생성할 수 있도록 돕는 AI 기반 클라우드 어시스턴트 서비스**입니다.

복잡한 AWS 콘솔 대신,

사용자는 단순히 그림을 그리거나 요구사항을 입력하는 것만으로

AI가 인프라 구조를 이해하고 Terraform 코드로 변환해줍니다.

---

# 2. 문제 정의 (The Problems)

### 1️⃣ AWS 진입 장벽 문제

- AWS는 설정이 복잡하고 학습 비용이 높음
- 초보자는 EC2 하나 띄우는 것도 어려움

### 2️⃣ 비용 예측 불가능

- 어떤 리소스를 쓰면 얼마가 나오는지 직관적으로 알기 어려움
- 실수로 리소스를 켜두면 요금 폭탄 발생

---

# 3. 해결 방법 (Solution)

Sketch-to-Cloud는

AI를 활용하여 다음과 같은 흐름을 제공합니다:

👉 입력 → 해석 → 구조화 → 코드 생성 → 시각화 → 비용 예측

즉,

**“아이디어 → 인프라 설계 → 코드”까지 자동화**하는 서비스입니다.

---

# 4. 핵심 기능 (Phase 1 - MVP)

## 🎯 목표

**“그림/텍스트 입력이 실제 인프라 설계 결과로 이어지는 Core Flow 완성”**

---

## 1️⃣ 스케치/텍스트 입력

- 사용자가 손그림 이미지 또는 텍스트로 요구사항 입력
- 예:
    - “EC2 2개 + RDS 하나”
    - 간단한 아키텍처 스케치

---

## 2️⃣ AI 아키텍처 분석

- Amazon Bedrock (Claude) 기반 분석
- 입력을 AWS 리소스 구조로 변환

예시 출력(JSON):

```json
{
  "vpc":true,
  "ec2":2,
  "rds":"mysql",
  "public":true
}
```

---

## 3️⃣ Terraform 코드 생성

- 분석 결과를 기반으로 Terraform 코드 자동 생성
- 실행 가능 수준이 아닌 **구조적으로 올바른 코드 생성**에 집중

---

## 4️⃣ 인프라 구성도 시각화

- React Flow 기반 다이어그램
- VPC, EC2, RDS 관계를 시각적으로 표현

👉 사용자 입장에서 가장 직관적인 결과 확인 가능

---

## 5️⃣ 예상 비용 산출

- 리소스 기반 월 예상 비용 계산
- 주요 비용 요소 분석 및 표시

예:

- EC2: ₩12,000
- RDS: ₩18,000
- 총합: ₩30,000

    👉 “이미지 업로드 + 텍스트 입력”만 사용

## 2️⃣ 비용 중심 설계

## 3️⃣ 시각화 중심 UX

# 8. 기술 스택 (Tech Stack)

## Frontend

- React
- Tailwind CSS
- React Flow (다이어그램 시각화)

## Backend

- Spring Boot (Java)

## AI

- Amazon Bedrock (Claude)

## Infrastructure

- Terraform
- AWS (VPC, EC2, RDS 중심)

## DevOps

- LocalStack (로컬 테스트)
- GitHub Actions

---

# 9. 시스템 흐름 (Core Flow)

```
[사용자 입력]
(이미지 / 텍스트)
        ↓
[AI 분석]
→ AWS 구조 JSON 생성
        ↓
[Terraform 코드 생성]
        ↓
[비용 계산]
        ↓
[프론트]
→ 다이어그램 + 코드 + 비용 표시
```
